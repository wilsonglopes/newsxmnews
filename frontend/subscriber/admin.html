📘 Tutorial completo — Conectar Facebook ao XIXO

🎯 O que você vai fazer
Criar um app no Facebook Developers, dar 3 permissões, gerar um token "eterno" da sua Página, e colar aqui no XIXO. Tempo estimado: 15 minutos.

📋 Antes de começar — você precisa:
  • Ser administrador da Página do Facebook que quer conectar
  • Ter conta no Facebook Developers (qualquer Facebook serve)
  • O ID da Página em mãos — veja como descobrir no fim do tutorial

PARTE 1 — Criar o App no Meta for Developers

  1. Acesse: developers.facebook.com/apps
  2. Clica no botão azul "Criar app" (canto superior direito).
  3. Detalhes do app:
    • Nome do app: algo como xmnews_post (sem espaços, só letras/números/underscore)
    • E-mail de contato: seu e-mail
    • Clica Avançar
  4. Casos de uso — aqui é a parte CRÍTICA:
    • No filtro à esquerda, clica em "Gerenciamento de conteúdo"
    • Vai aparecer 5 opções. Marca o checkbox de "Gerenciar tudo na sua Página" (ícone de bandeira azul)
    • Clica Avançar
  5. Empresa: escolhe "Eu mesmo" ou cria um perfil empresarial básico. Avançar.
  6. Requisitos: só clica Avançar. Não precisa preencher nada.
  7. Visão geral: clica Criar app ou Concluir.
✅ Resultado da Parte 1: Você está agora no painel do seu app. Anota: no canto superior você vê o nome do app.

PARTE 2 — Habilitar as 3 permissões

  1. No painel do app (você está aí depois de criar), procura a seção "Personalização do app e requisitos".
  2. Clica na primeira linha: "Personalizar o caso de uso 'Gerenciar tudo na sua Página'" (com a setinha à direita).
  3. Vai abrir uma tela com várias permissões listadas. Vai aparecer um botão "+ Adicionar" ao lado de cada uma.
  4. Adiciona SOMENTE estas 3: (ignore as outras como pages_manage_metadata ou pages_read_user_content)
    • pages_show_list — listar suas Páginas
    • pages_manage_posts — publicar posts
    • pages_read_engagement — ler dados
  5. Cada permissão adicionada vira "Pronto para teste" com botão "Ações" do lado.
✅ Resultado da Parte 2: 3 permissões adicionadas, status "Pronto para teste". Sem precisar de App Review da Meta enquanto for sua Página.

PARTE 3 — Gerar o User Token (curto)

  1. Abre em nova aba: Graph API Explorer
  2. No painel à direita, configura:
    • App da Meta: seleciona o app que você acabou de criar
    • Usuário ou Página: deixa "Token do usuário"
  3. Em Permissões (também no painel direito), clica em "Adicionar uma permissão". Marca as MESMAS 3:
    • pages_show_list
    • pages_manage_posts
    • pages_read_engagement
  4. Clica no botão azul "Generate Access Token". Vai abrir um popup do Facebook pedindo autorização.
  5. ⚠️ ARMADILHA #1 — Selecionar a Página no popup
    No popup do Facebook ele vai perguntar "Quais Páginas você quer permitir acesso?":
    • NÃO clica em "Pular nesta etapa" ou "Optar por todas as Páginas atuais e futuras"
    • Marca SOMENTE a Página específica que você quer conectar
    • Continua, aceita as 3 permissões, finaliza
    Se errar e pular, o token gerado não vai ter acesso à Página. Volta e refaz.
  6. O campo "Token de acesso" agora tem o User Token. Esse vence em ~1 hora — vamos prolongar.

PARTE 4 — Estender o token para 60 dias (Long-Lived)

  1. Copia o User Token do passo anterior (do campo "Token de acesso" no Graph Explorer).
  2. Abre em nova aba: Access Token Debugger
  3. Cola o token no campo, clica Depurar.
  4. Confirma:
    • Tipo: User
    • Escopos: tem as 3 permissões
    • Escopos granulares: sua Página aparece listada
    Se não tiver, volta e refaz a Parte 3.
  5. Rola pra baixo, clica no botão "Estender token de acesso".
  6. ⚠️ ARMADILHA #2 — Qual token copiar
    Vai aparecer uma caixa nova embaixo da tela, com o título "Este novo token de acesso de vida longa expirará em X dias".
    Copia o token de DENTRO dessa caixa nova (não o do topo da página!). Esse é o Long-Lived User Token (~60 dias).

PARTE 5 — Pegar o Page Token "eterno"

🔑 Conceito-chave: O User Token identifica VOCÊ (vence em 60 dias).
O Page Token identifica a PÁGINA (NÃO EXPIRA, se derivado de um Long-Lived User Token).
Pra publicar em Página precisamos do Page Token.
  1. Volta pro Graph API Explorer.
  2. ⚠️ ARMADILHA #3 — Trocar o token no campo
    O campo "Token de acesso" pode ter regenerado um token CURTO automaticamente (acontece se trocou de aba).
    Apaga totalmente o campo e cola o Long-Lived (que você copiou na Parte 4).
  3. Verificação importante: abre o Debugger em outra aba, cola o token que você acabou de pôr no Explorer, e confere:
    • Expira: deve mostrar "aproximadamente 60 dias" ou similar
    • Se mostrar "em cerca de 1 hora" — é o CURTO, não funciona. Volta à Parte 4 e refaz.
  4. Com o Long-Lived no campo, agora monta o endpoint. Na barra de URL do Explorer (campo grande do meio, depois de "v25.0 /"), digita: 
    ID_DA_PAGINA?fields=access_token,name
    Ex: 1052179677989730?fields=access_token,name
  5. Clica Enviar (botão azul à direita). A resposta vai aparecer no centro: 
    {
  "access_token": "EAAf7I3r8eGs...",  ← ESSE É O PAGE TOKEN
  "name": "Nome da Página",
  "id": "1052179677989730"
}
  6. Copia o valor inteiro de access_token (sem as aspas).
  7. Confirme que NÃO expira: cola esse Page Token no Debugger (em outra aba) e verifica:
    • Tipo: Page
    • Expira: Nunca ✅
    Se mostrar "em cerca de 1 hora", o token no Explorer não estava Long-Lived — volta à Parte 5 passo 2.

PARTE 6 — Como descobrir o ID da Página

Existem 3 formas:

  1. Pelo Facebook: entra na sua Página → menu lateral → "Sobre" → seção "Transparência da Página" ou "Mais informações" → o ID está listado.
  2. Pelo Business Suite: Configurações → Página → ID aparece nas informações.
  3. Pelo Graph Explorer: com o Long-Lived User Token no campo, faz me/accounts. A resposta lista todas suas Páginas com IDs (mesmo que liste vazio em alguns casos, o método pelo ID direto da Parte 5 sempre funciona).

📷 PARTE EXTRA — Conectar Instagram (opcional)

🎯 O que faz: além de postar no Facebook, posta automaticamente no Instagram (foto única no feed) com a mesma legenda. Usa o MESMO Page Access Token — não precisa de token separado.
  1. Pré-requisito: sua conta Instagram precisa ser Profissional (Business ou Creator) e estar conectada à Page do Facebook.
    • No app do Instagram → Perfil → ☰ → Configurações → Tipo de conta → "Mudar para conta profissional"
    • Depois: Configurações → Meta → Contas conectadas → conectar à Page do FB
  2. Adicionar permissões IG no app Meta — volta no Meta for Developers, abre seu app:
    • Vá no caso de uso "Gerenciar tudo na sua Página" (mesma da Parte 2)
    • Procure e adicione com "+ Adicionar":
      • instagram_basic
      • instagram_content_publish
  3. Regerar Page Token com as novas permissões — refaz as Partes 3, 4 e 5 do tutorial.
    ⚠️ Importante: no Graph API Explorer, marca também instagram_basic e instagram_content_publish em "Permissões" antes de "Generate Access Token". Senão o Page Token novo continua sem acesso ao IG.
  4. Atualizar o Page Token aqui no XIXO:
    • Sites → Editar XMNews → cola o token NOVO no campo "Page Access Token"
    • Clica 🔌 Testar conexão
    • Se o Instagram estiver conectado à Page e o token tiver as permissões IG, vai aparecer um bloco rosa embaixo: "📷 Instagram detectado: @xmnews"
    • Marca o checkbox "Publicar também no Instagram"
    • Salva
✅ Pronto! A partir de agora, cada post no Facebook irá automaticamente também pro Instagram.

PARTE 7 — Configurar aqui no XIXO

  1. Fecha este tutorial.
  2. Marca o checkbox "Ativar publicação no Facebook para este site".
  3. No campo "ID da Página": cola o ID (só os números).
  4. No campo "Page Access Token": cola o Page Token (o "eterno" da Parte 5).
  5. Clica 🔌 Testar conexão:
    • ✓ Verde: "Conectado à página: NOME" — está OK!
    • ✗ Vermelho: confere ID e token; talvez o token tenha expirado e precise refazer
  6. Na seção Publicação automática, marca o checkbox 📘 nas fontes que devem postar no Facebook automaticamente quando rodar o autopub.
  7. Clica Salvar site.
✅ Pronto! O que acontece agora:
  • Cada publicação manual ("Publicar este") terá um checkbox "Publicar também no Facebook" marcado por padrão
  • O autopub vai gerar um card visual e postar na Página, com link da matéria
  • Cada post vai pra Página com o card 1080×1080 + texto + URL da matéria

🆘 Resolução de problemas

Erro: "Session has expired"
O User Token usado pra gerar o Page Token era curto (1h). Refaz a Parte 4 com cuidado, garantindo copiar o Long-Lived. Refaz a Parte 5.
Erro: "publish_actions are not available. It has been deprecated"
Você colou um User Token no lugar do Page Token. Eles parecem iguais (ambos começam com EAA) mas são diferentes. Volte à Parte 5 e copia o access_token de DENTRO do JSON da resposta.
/me/accounts retorna lista vazia
Acontece com Páginas "Nova Experiência" ou via Business Manager. Use o método da Parte 5 (pelo ID direto), que sempre funciona.
Page Token mostra "Expira: em 1 hora" no Debugger
O token no Explorer não era Long-Lived quando você consultou o ID. Garanta que copiou e colou o Long-Lived ANTES de fazer o GET no ID da Página.
Após algum tempo o token parou de funcionar mesmo sendo "eterno"
Page Tokens duram pra sempre, mas podem ser invalidados se: a senha do Facebook do admin foi alterada; o app foi revogado nas Configurações de Segurança do Facebook do admin; o admin perdeu o role na Página. Refaça as Partes 3-5.
